# Publication Checklist

## Safe To Post

- `README.md`
- Files in `docs/`
- `linkedin-project-description.md`
- `linkedin-progress-post.md`
- Selected screenshots in `screenshots/`
- High-level descriptions of the lab architecture, tools, schema design, ERD, constraints, indexes, and learning goals

## Intentionally Excluded

- Private `evidence/` folder
- Raw command output logs
- System inventory files
- Local IP addresses and private network details
- Hostnames and unnecessary machine identifiers
- Credential-related files
- Password files
- Private keys
- Environment files
- Backup archives
- Raw schema dump files
- Raw CSV exports
- Full backup/tool evidence
- Connection strings with credentials

## Needs Human Review Before Publishing

- Open each screenshot and confirm it does not show private account names, private paths, IP addresses, credentials, or unrelated personal information.
- Review LinkedIn text for tone and accuracy.
- Confirm the project is still described as in progress.
- Confirm no production claims were added.
- Confirm no private source evidence was copied into this folder.
- Confirm `.gitignore` is present before initializing or pushing a Git repository.
